# Main Block — MCU, Power Path, Battery, and System Integration

## Scope

This document covers all hardware subsystems not addressed by the
ADC and DC-DC blocks:

- STM32F103C8T6 microcontroller and pin assignment
- Power-on sequencing (LTC2954 power-on controller)
- Digital 3.3 V LDO (LP5907)
- Li-ion battery system (18650 cell, TP4056 charger, DW01A BMS)
- OLED display (SSD1306)
- User input (button, rotary encoder)
- 4-wire Kelvin test terminals
- USB-C input and ESD protection
- PCB layout, mechanical, and thermal integration

Detailed current-source analysis (OPA333, IRLR7843, INA226 filters)
is in the ADC block. Boost converter design (MT3608, inductor,
output filtering) is in the DC-DC block.

Firmware architecture, calibration procedures, and UI screens are
documented separately in the firmware folder.



## STM32F103C8T6 — Pin Assignment

The MCU is the central controller: it generates PWM, drives I2C
peripherals, manages battery monitoring, and holds the "firmware
alive" feedback to the LTC2954 power-on controller.

### Active Pin Map

| Pin    | Function          | Direction | Description                                |
|--------|-------------------|-----------|--------------------------------------------|
| PA7    | ADCBAT            | Input     | Battery voltage via 100k/330k divider      |
| PA8    | PWM_OUT           | Output    | 100 kHz PWM → RC filter → OPA333 reference |
| PA13   | SWDIO             | I/O       | SWD programming data, 120 Ω series R       |
| PA14   | SWCLK             | Input     | SWD programming clock, 120 Ω series R      |
| PC13   | LED_STATUS        | Output    | "Firmware is alive" indicator LED          |

### PA7 — Battery Voltage Monitoring (ADCBAT)

Battery voltage is divided to fit the 3.3 V ADC range.
Divider: 100 kΩ (top) / 330 kΩ (bottom) from VBAT to GND.

    V_adc = V_bat × R_bottom / (R_top + R_bottom)
          = V_bat × 330k / (100k + 330k)
          = V_bat × 0.767

Range mapping:
- V_bat = 4.2 V (full) → V_adc = 3.22 V (just under 3.3 V ref)
- V_bat = 3.0 V (cutoff) → V_adc = 2.30 V
- V_bat = 2.5 V (BMS cutoff) → V_adc = 1.92 V

Resistor values are chosen high (430 kΩ total) to minimize
dividing current draw from the battery. A 1 µF X7R capacitor
is placed in parallel with the 330 kΩ (R47) for ADC input
buffering and noise rejection on PCB-level switching events
(at 1.2 MHz from MT3608 a smaller cap is insufficient).

ADC sampling timing: programmed to read ADCBAT once per
measurement cycle, with internal averaging configured
to suppress converter switching noise.

### PA8 — PWM_OUT (100 kHz)

PWM frequency is 100 kHz, generated by TIM1_CH1. The signal feeds
the two-stage RC filter documented in the ADC block; the cutoff
frequency (159 Hz) gives ~-40 dB/dec attenuation of the PWM
fundamental.

Duty cycle is set in firmware to select between 100 mA, 500 mA,
and 1000 mA test current ranges. PWM up-count, center-aligned
mode is supported but not strictly required for RC settling at
this frequency.

### PA13 / PA14 — SWD Programming Header (Debug)

SWDIO and SWCLK lines are routed through 120 Ω series resistors.
This serves two purposes:
- Limits inrush current during programming hot-plug events
- Provides a degree of ESD isolation from external programmers
  before the TVS array settles

A 4-pin SWD header (VCC, SWDIO, SWCLK, GND) is placed on the
board edge. In production these pads are unrouted or covered
with conformal coating if no calibrator access is required.

### PC13 — Firmware Status LED

PC13 is used as a heartbeat indicator. The LED is driven low-active
(opens when the firmware loop is running). It is intentionally
NOT a power indicator — power state is visible via the OLED.

Series R = 1 kΩ with a standard red LED (V_F ≈ 2.0 V, I_F ≈ 1 mA):

    I_F = (3.3 - 2.0) / 1k = 1.3 mA

PC13 has the limitation that the LED remains in an
"inconsistent" state during and after reset, but for this
project on-board reset is not a user-visible event — the
100 ms lamp test during boot is acceptable.

### PC14 / PC15 — HSE Crystal (8 MHz)

External 8 MHz HSE crystal drives the PLL to produce the
72 MHz SYSCLK. Crystal load capacitor selection:

    C_load = 2 × (C_L − C_stray) = 2 × (20pF − 5pF) = 30 pF

C17 = C18 = 30 pF (C0G/NP0 dielectric).

HSE is preferred over the internal HSI (±1% accuracy) because
PWM frequency error at ±1% directly maps to test current error
of ±1% — at µΩ measurement levels this dominates accuracy budget.

The crystal is placed close to the OSC pins with a guarded
GND loop around it to prevent 1.2 MHz MT3608 switching noise
from coupling into the oscillator load capacitance.

### PB0 — USB Power Detect (STATUS_CHARGE)

PB0 reads the presence of 5 V on USB-C VBUS through a
47 kΩ / 82 kΩ divider.

    V_pb0 = 5.0 × 82k / (47k + 82k) = 5.0 × 0.636 = 3.18 V

This sits below 3.3 V MCU supply and is directly read as a
GPIO high → "USB connected" condition. Reverse-biased action
is not relevant — the divider is unconditionally high impedance
and draws < 100 µA from VBUS even when the device is off.

STATUS_CHARGE is sampled by the firmware to enable battery
voltage display mode on OLED and to detect charging cycles.

### PB1 — Firmware Feedback to LTC2954

PB1 is the firmware-running feedback signal to LTC2954. While
the firmware is alive (firmware loop is in normal state), PB1
is driven HIGH, holding the LTC2954 EN output active so that
the boost converter remains enabled.

If the firmware hangs (e.g. infinite loop, HardFault without
recovery), PB1 drops, LTC2954 KILL timing fires, and the system
shuts down cleanly via Q1.

This provides a hardware WatchDog independent of the internal
IWDG. For belt-and-braces security, the firmware also runs an
IWDG that, on timeout, intentionally drives PB1 low so that
the hardware WatchDog also triggers.

### PB2 / BOOT0 — Boot Mode Selection

Both PB2 (BOOT1) and BOOT0 are pulled down to GND via 100 kΩ
resistors, selecting boot from internal flash (BOOT1 = 0,
BOOT0 = 0).

The 100 kΩ value is standard practice when no external boot
source is connected — it draws no DC current except negligible
leakage and pulls the line to a defined state even if the SWD
header is floating.

### PB6 / PB7 — I2C Bus (Standard Mode)

I2C1 operates at Standard Mode (100 kHz) — sufficient for
INA226 (512 conversions/sec) and SSD1306 refresh rates.

Pull-ups are 4.7 kΩ to 3.3 V on both SCL and SDA. Selected
per I2C specification for Standard Mode:

    R_min = (V_cc − V_ol_max) / I_ol = (3.3 − 0.4) / 3mA = 967 Ω
    R_max = 1000 ns / (C_bus + C_pins) ≈ 10 kΩ at 100 kHz

A 4.7 kΩ midpoint value gives clean edges with the
combined bus capacitance of INA226 + SSD1306 + trace
(estimated C_bus < 50 pF).

If firmware debugging reveals timing-dependent issues
during long cable I2C runs to the OLED, switching to Fast
Mode (400 kHz) with 2.2 kΩ pull-ups is a trivial firmware-side
change with no hardware rework required.



## LTC2954 — Power-On Controller

LTC2954 manages push-button power-on/off. The MCU alone cannot
latch its own power supply — once the user releases the button,
VBAT to the boost converter would drop. LTC2954 provides
hardware-latched EN that keeps Q1 conducting until firmware
orderly issues a shutdown.

### Sequencing

1. User presses button (≈ 100 ms debounced) →
   LTC2954 drives EN high → Q1 (AO3401A P-channel MOSFET) conducts
   → MT3608 boost starts → LP2985 / LP5907 come up → STM32 boots
2. STM32 firmware initializes → drives PB1 HIGH within first 100 ms
3. After this point the firmware "owns" the power state
